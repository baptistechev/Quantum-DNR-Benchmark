# Quantum DNR Benchmark

Benchmark suite for Distribution Network Reconfiguration (DNR) comparing classical, quantum simulation, and quantum hardware (QPU) approaches using the TrOMA framework.

## Repository structure

```
data/                          # Static reference data
  {9,12,15,33}_bus.csv         # Brute-forced objective landscapes (for percentile rank evaluation)

benchmark_scripts/
  benchmark_simulation.py      # Main benchmark: state-of-the-art + MCCO classical & quantum simulation methods
  benchmark_analytical.py      # Deterministic analytical methods (Baran, Merlin, Taylor)
  benchmark_qpu.py             # Quantum hardware runs on IBM QPU or Noisy simulator (Default: ibm_fez)
  seeds.jsonl                  # Seed that were used for the figures
  sim_time.json                # Per-method timing estimates (used to budget runs)

results/                       # Benchmark outputs (JSONL)
  benchmark_runs.jsonl         # Output of benchmark_simulation.py
  analytical_runs.jsonl        # Output of benchmark_analytical.py
  qpu_runs.jsonl               # Output of benchmark_qpu.py (shipped as-is, see below)
  results_analysis.jsonl       # Aggregated analysis (generated by the notebook)

figures/                       # Generated by res_analysis.ipynb
  fig*.pdf                     # All figures
  table_benchmark.tex          # LaTeX table

res_analysis.ipynb             # Analysis notebook: reads results/, produces figures/
pyproject.toml                 # Python package definition and dependencies
```

## Installation

Requires Python >= 3.11 and a CUDA-capable GPU for quantum simulation.

```bash
git clone <this-repo>
cd Quantum-DNR-Benchmark
pip install -e .

# For GPU-accelerated simulation (requires CUDA 11 environment):
pip install -e ".[gpu]"
```

Several dependencies are installed from source branches:
- [EasyDNR](https://github.com/baptistechev/EasyDNR) — DNR network modeling
- [TrOMA](https://github.com/baptistechev/TrOMA) (`beta` branch) — combinatorial optimization framework
- [Qamomile](https://github.com/baptistechev/Qamomile) (`feature/alternating_ansatz` branch) — quantum ansatz
- [DNRlib](https://github.com/baptistechev/DNRlib) (`fix/taylor` branch) — classical DNR solvers

## Reproducing the results

The results in `res_analysis.ipynb` come from three benchmark scripts run in sequence.
Each script writes a timestamped JSONL file into `results/`; rename its output to the
canonical filename that the notebook expects.

### Step 1 — Analytical methods (seconds)

```bash
python benchmark_scripts/benchmark_analytical.py
# Produces: results/analytical_runs_<timestamp>.jsonl
mv results/analytical_runs_*.jsonl results/analytical_runs.jsonl
```

### Step 2 — Simulation benchmark (hours, requires GPU)

The simulation benchmark runs up to 30 runs per method, budgeted by estimated wall-clock time.

**Fresh run** (generates new random seeds):

Uncomment all methods in the `sim_types` dict inside `benchmark_simulation.py`, then:

```bash
python benchmark_scripts/benchmark_simulation.py
# Produces: results/benchmark_runs_<timestamp>.jsonl
mv results/benchmark_runs_*.jsonl results/benchmark_runs.jsonl
```

**Reproducible re-run** (same seeds as the published results):

```bash
python benchmark_scripts/benchmark_simulation.py --run-from-seeds benchmark_scripts/seeds.jsonl
```

This replays the exact same number of runs with the exact same seeds per method. The `seeds.jsonl` file was extracted from the original benchmark output.

Useful flags:
- `--networks 9_bus 15_bus` — run only specific networks
- `--methods classical quantum` — run only specific categories
- `--estimate` — print estimated run counts and durations without running
- `--sanity-only` — run a quick sanity check and exit
- `--sim-time-update` — re-measure per-method timing and update `sim_time.json`

### Step 3 — QPU benchmark (requires IBM Quantum access)

QPU results depend on quantum hardware and are **not reproducible** — hardware noise varies between runs and the device calibration changes daily. The file `results/qpu_runs.jsonl` is shipped with the repository and can be used directly.

To re-run on hardware:

```bash
export IBM_QUANTUM_TOKEN='your-token-from-quantum.ibm.com'
python benchmark_scripts/benchmark_qpu.py
# Produces: results/qpu_runs_<timestamp>.jsonl
mv results/qpu_runs_*.jsonl results/qpu_runs.jsonl
```

### Step 4 — Analysis notebook

Once all three JSONL files are in `results/`, run the notebook from the repo root:

```bash
jupyter notebook res_analysis.ipynb
```

Run all cells to regenerate figures (`figures/fig*.pdf`), the LaTeX table (`figures/table_benchmark.tex`), and the intermediate analysis files.

## Hardware requirements

- **RAM**: >= 16 GB
- **GPU**: NVIDIA GPU with CUDA 11 support (for `qiskit-aer-gpu-cu11`)
- **CPU**: multi-core recommended (simulation uses all available cores)

The 33-bus quantum simulations use matrix product state (MPS) mode and are the most memory-intensive.
